### D.28.b — Port `training()` method (≈ 2-3 h)
**Goal :** add `AGLTrainer.training(n_epochs, batches=None)` matching student `training()` L904-1035.

**Key student behaviors to preserve :**

1. **`meta = False` override bug at student L969.** Student overrides the `meta` parameter
   regardless of what's passed, so in training() phase the 2nd-order network is **never
   back-propagated**. It still does a forward pass to produce `output_second_order` and
   compute `loss_2` (for logging only).
   - Port should replicate this bit-exactly. Add a comment + flag `# student L969 bug — 2nd-order
     frozen in training phase`.
   - Parametrize via `self.cfg.train.train_meta_override: bool` (default `False`, student behavior)
     so a future experiment can flip it to `True` for exploration.

2. **Grammar A patterns at each epoch** : `patterns_tensor = Array_Words(2, 45*factor)` — fresh
   draw, never reused. Port uses `generate_batch(GrammarType.A, 45*factor)` each epoch.

3. **Cascade logic identical to pre_train** — same `cascade_iters` / `cascade_rate` from setting.

4. **Loss_2 computed but not backpropagated (meta=False branch)** : student L1004
   `loss_2 = networks[index][3](output_second_order, order_2_tensor)` — no `.backward()`,
   no step. Port logs it in `losses_2[epoch]` but doesn't apply.

5. **Loss_1 always back-propagated** via `networks[index][2](W, patterns, h2, h1, lam)` with
   num_args=5 path (CAE_loss).

6. **Scheduler steps only on optim_1** in the else branch (L1019-1020).

7. **Per-epoch precision logged via `calculate_metrics`** (L1025) — precision-only, inlined
   as in port's `evaluate()`.

**Method signature :**
```python
def training(
    self,
    n_epochs: int | None = None,
    *,
    batches: list[TrainingBatch] | None = None,
) -> tuple[np.ndarray, np.ndarray, np.ndarray]:
    """Run the Grammar-A training phase (student L904)."""
```
Returns : `(losses_1, losses_2, precision)` — each 1-D over epochs.

**Verify :**
- Smoke : `trainer.pre_train()` then `trainer.training(n_epochs=12)` then `trainer.evaluate()`.
  Check loss curves are sane (decreasing for loss_1, roughly flat for loss_2).
- Parity test (new) : pin RNG, compare port-training vs student-training loss_1 sequence.

**DoD :** `training()` method added, docstring cites student L904, parity test passes, 1 commit.

### D.28.c — Port `create_networks()` 20-copy replication (≈ 1-2 h)
**Goal :** implement replication so `AGLTrainer` can orchestrate 20 networks per seed.

**Design :** rather than carrying a list-of-20 inside `AGLTrainer`, extract the
replication into a new `AGLNetworkPool` class :

```python
@dataclass
class NetworkCell:
    first_order: FirstOrderMLP
    second_order: SecondOrderNetwork
    optim_1: Optimizer
    optim_2: Optimizer
    sched_1: StepLR
    sched_2: StepLR

class AGLNetworkPool:
    def __init__(self, trainer: AGLTrainer, num_networks: int):
        """Create num_networks copies of trainer's networks (post-pretrain weights)."""
        self.cells: list[NetworkCell] = [...]

    def train_range(self, start: int, end: int, n_epochs: int) -> dict:
        """Run training() on cells [start:end] for n_epochs."""

    def evaluate_all(self) -> list[dict]:
        """Evaluate every cell, return per-cell metrics."""
```

Alternatively, keep it flat on `AGLTrainer` with a `self.network_pool` list. Simpler but
less modular. **Prefer `AGLNetworkPool` class** for cleanliness + testability.

**Weight copy mechanism :** student uses `torch.save(PATH) → torch.load(PATH)` which is disk-round-
trip. Port uses `copy.deepcopy(state_dict())` → `load_state_dict()` — equivalent, no disk I/O.

**Verify :**
- Unit test : create pool, verify each cell has independent parameters (modifying one doesn't
  affect another).
- Integration : 20-cell pool, pretrain → replicate → train(0,10,12) + train(10,20,3) → eval.

**DoD :** `AGLNetworkPool` class + tests + 1 commit.

### D.28.d — Refactor `evaluate()` for network list (≈ 1 h)
**Goal :** `AGLTrainer.evaluate(pool: AGLNetworkPool) → dict` aggregates per tier.

**Changes to `trainer.py:evaluate` L325-412 :**
- Accept a pool.
- For each cell in pool, run forward cascade on fresh Grammar A + B test batch (size
  `len(pool) × factor` = 20 per grammar = 40 total, per student L1197-1198).
- Compute per-cell `precision_1st`, `wager_accuracy` (TP+TN)/(TP+FP+TN+FN), plus
  `precision_2nd`, `recall_2nd`, `f1_2nd` (new metrics we drop in port currently).
- Aggregate :
  - `high` tier = mean/std over cells `[0 : num_networks//2]`.
  - `low` tier = mean/std over cells `[num_networks//2 : num_networks]`.
  - `overall` = mean/std over all cells.
- Return nested dict :
```python
{
  "high": {"precision_1st": ..., "precision_1st_std": ..., "wager_accuracy": ..., ...},
  "low":  {...},
  "overall": {...},
}
```

**Fix the incorrect docstring in current evaluate (L343-346)** — "post-hoc seed-pool split" is
**wrong**. High/Low is per-seed, 10 cells × 12 epochs vs 10 cells × 3 epochs.

**Verify :**
- Parity test vs student `testing()` on fixed seeds + fixed network pool.
- 1 commit.
